No construirías una casa sin un arnés de seguridad. No codees sin [control de versiones](https://github.com/resources/articles/software-development/what-is-version-control).

## Sistemas de Control de Versiones

Git domina el mercado de VCS, con los demás relegados a casos legacy.

| Sistema                                     | Superpoder                     | Defecto                          |
| ------------------------------------------- | ------------------------------ | -------------------------------- |
| [Git](https://git-scm.com/)                 | Distribuido, magia de branches | Curva de aprendizaje pronunciada |
| [SVN](https://subversion.apache.org/)       | Simplicidad centralizada       | Sin commits locales              |
| [Mercurial](https://www.mercurial-scm.org/) | Más fácil que Git              | Perdió las guerras de VCS        |
| [CVS](https://cvs.nongnu.org/)              | Pionero en versionado          | Tracking a nivel de archivo      |

## Tips de Git

1. **Git primero, código después**: Siempre hacé `git init` antes de escribir la primera línea de código.
2. **Consola > GUI**. Herramientas como [GitHub Desktop](https://desktop.github.com/download/) están bien, pero aprendé estos cuatro comandos primero:

   ```bash
   git add .
   git commit -m "save point"
   git push
   git checkout -b
   ```

   ¿Por qué? Porque cuando el plugin de Git de tu IDE se rompa misteriosamente (va a pasar), vas a necesitar debugear como un pro.

3. **Las reglas de tu equipo > gitflow de libro**: Quizás te encanten las feature branches, pero cada empresa tiene su workflow "sagrado". Observá primero, después sugerí mejoras.
